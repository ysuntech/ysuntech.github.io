使用hugo搭建博客系统

介绍

作为web技术从业人员，经常有一些笔记心得需要记录。可以选择在csdn或者oschina等站点写博客。也有人购买VPS，用wordpress搭建自己的博客。这样比较自由，但购买VPS，搭建lamp环境需要一定的Time和Money。其实有更简单的解决方案：使用hugo和github pages搭建全静态免费博客。

搭建博客

让我们开始吧,按步骤来：

  1. 下载hugo软件，图省事可以直接下载二进制版本

  2. 将下载得到的hugo二进制文件加入PATH环境变量。这样我们可以从命令行输入”hugo”随时调用hugo

  3. 初始化博客项目

    cd ~
# 创建名为hugo_blog的博客站点
hugo new site hugo_blog

  4. 设置主题，配置站点

    cd hugo_blog/themes
# 这里我们使用beg 主题
git clone https://github.com/dim0627/hugo_theme_beg.git beg
# 修改配置文件
vim hugo_blog/config.toml

    修改内容

    languageCode = "zh-CN"
baseurl = "https://mytestaccout.github.io/"
title = "我的博客小站"
theme = "beg"
#googleAnalytics = "UA-XXXXXXXX-XX" # Optional
#disqusShortname = "XYW"
[params]
dateformat = "Jan 2, 2006" # Optional

    其中mytestaccout需要替换为自己的github账号

  5. 写一篇博客

    hugo new mydir/post1.md

    这样会生成content/mydir/post1.md 使用编辑器打开md文件,文件分为两部分，上半部分为yaml格式，下半部分为文章内容。 yaml部分可以设置标题(title)，日期(date)， 是否草稿(draft)，标签(tags)，分类(categories)。格式大致如下：

    ---
title: "XXX调度系统:分布式和常见问题"
date: 2017-10-20T16:48:32+08:00
draft: false
tags: ["XXX标签","调度系统"]
categories: ["大数据"]
---
以下是文章内容，markdown格式

    注意默认设置下，draft为false的文章才能被预览和发布

  6. 本地预览

    hugo server
# 然后打开 http://localhost:1313/

  7. 将md发布为html到public目录

    hugo

  8. 在github上用自己的账号创建名为:“{{github账号}}.github.io”的repo

  9. 将public目录下的文件push到刚创建的repo

  10. 打开浏览器，访问: https://{{github账号}}.github.io/ 我们的免费博客上线啦

补充说明

可以给自己的博客绑定域名(CNAME到{{github账号}}.github.io)

不付费的话github.com上的repo只能为public，我们可以将自己的hugo站点(包括md文件)提交到oschina上的git
